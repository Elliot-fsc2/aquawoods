# Aquawood / Schema Spark — Agent Guide

## Architecture

- **Dual routers**: TanStack Router (outer, SSR) → `AppShell` → react-router-dom `BrowserRouter` (inner). The `mounted` guard (`src/app/AppShell.tsx:15-17`) delays `BrowserRouter` mount until after TanStack Router settles to avoid `history.replaceState` conflicts.
- **Catch-all**: TanStack Router's `/$` route (`src/routes/$.tsx`) passes all URLs to the inner react-router app.
- **AppShell** wraps `AppProvider → ToastProvider → BrowserRouter → DesktopApp | MobileRoutes`.
- **Mobile detection**: `useMobileView()` hook switches between `MobileRoutes` (bottom tabs) and `DesktopApp` (sidebar).
- **No SSR data fetching** — TanStack Start SSR shell is empty; all data hydrates client-side from Supabase.

## Auth / Role System

| Role | Routes | Guard Component |
|------|--------|----------------|
| `admin` | All dashboard + Settings | `AdminOnly` |
| `employee` | Dashboard (no Settings) | `Protected` |
| `guest` | `/account/*` | `GuestProtected` |
| unauthenticated | `/login`, `/guest-login`, `/` | `PublicOnly`, `GuestPublic` |

Guards are in `src/app/DesktopApp.tsx` (lines 25-61) and `src/app/mobile/MobileRouter.tsx`.

## Data Flow

- **All live data from Supabase** — `src/app/data/mockData.ts` exports only TypeScript types and empty arrays (no seed data).
- **Realtime subscriptions** on `rooms`, `reservations`, `guest_requests`, `emergency_alerts`, `crm_guests` via Supabase Realtime.
- **Guest booking → staff reservation**: `addGuestBooking()` inserts into `guest_bookings`; a DB trigger (`mirror_guest_booking_to_reservation()`) mirrors it to `reservations`.
- **Staff auth**: Supabase Auth with `onAuthStateChange` → `loadUser()` reads `user_roles` and `profiles` tables.
- **Guest auth**: `loginGuest()` → `supabase.auth.signInWithPassword()`, then reads `guest_users` table.

## Supabase Gotchas

- **`guest_users` table is not in generated TypeScript types** (`src/integrations/supabase/types.ts`) — all `supabase.from("guest_users")` calls produce TS errors but work at runtime. Do not "fix" by removing calls.
- **18 SQL migrations** in `supabase/migrations/`; `supabase/all_migrations.sql` is a concatenated copy for one-shot deployment.
- **Storage**: `room-images` bucket; upload via `uploadRoomImage()` (max 5 MB, image-only).
- **Required env vars** (in `.env`): `SUPABASE_URL`, `SUPABASE_PUBLISHABLE_KEY`, `VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`. Service role key is optional (needed only for admin server functions).

## Known Quirks

- `dailyMetrics` in `Overview.tsx` must be imported from `../data/mockData` (empty array — fine for initial render; chart updates when Supabase data arrives).
- `bookingSources` in `Overview.tsx` and `Reports.tsx` is hardcoded static data, not from Supabase.
- `MobileStaffPortal` (`src/app/mobile/MobileStaffPortal.tsx`) uses `StaffGuard` wrapper — visible to both admin and employee roles.
- `Settings` page is wrapped with `AdminOnly` — only admin role can access it.
- The app was built for a single property (Aquawood Garden Resort) — property name/logo can be changed in-app.
